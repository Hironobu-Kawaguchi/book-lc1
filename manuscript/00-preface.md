# まえがき

## なぜこの本を書いたのか

生成AIの急速な進化により、企業における情報活用の方法が根本的に変わりつつあります。
ChatGPT の登場から2年あまりで、LLM を活用したアプリケーション開発は実験的な段階から、実用的なビジネスツールへと進化しました。
しかし、単なる質問応答システムから一歩進んで、複雑なタスクを自律的に遂行する「エージェント」や、企業の知識ベースを活用する「RAG」を実装しようとすると、多くの課題に直面します。

本書は、そうした課題に取り組むエンジニアとビジネスパーソンのために書かれました。
LangChain と LangGraph は、エージェント型アプリケーションと RAG システムを構築するための強力なツールですが、その豊富な機能と頻繁なアップデートは、初学者にとって障壁となることもあります。
特に、2024年から2025年にかけて両ライブラリが 1.0 版に到達し、API が大きく変更されたことで、過去の情報が使えなくなったケースも少なくありません。

本書では、LangGraph 1.0 と LangChain 1.0 を軸に、エージェントと RAG の設計と実装を体系的に解説します。
単なる API リファレンスではなく、実際のプロジェクトで使える実装パターン、ビジネス視点での企画方法、運用時の注意点まで、実践的な知識を提供します。

## 本書の特徴

本書は以下の3つの特徴を持ちます。

### 1. LangChain 1.0 / LangGraph 1.0 に完全対応

本書のすべてのコード例は、LangChain 1.0 と LangGraph 1.0 の API を使用しています。
0.x 系から 1.0 系への移行方法も解説しており、既存プロジェクトの更新にも役立ちます。

### 2. ビジネスと技術の両面をカバー

技術的な実装だけでなく、ビジネス視点でのサービス企画、KPI 設定、ROI 計算なども扱います。
エンジニアとビジネスパーソンが協力してプロジェクトを進められるよう、両者の視点を往復しながら解説します。

### 3. 実践的なコード例とプロジェクト構成

本書のサンプルコードは、GitHub リポジトリの `tutorial/` ディレクトリに格納されています。
uv を使った環境構築から、FastAPI を使ったサーバー実装、Docker 化、AWS へのデプロイまで、実際のプロジェクトで使える構成を示します。

## 対象読者

本書は以下のような方々を対象としています。

- AI を活用した新規事業や業務改善を検討しているビジネスパーソン
- LLM アプリケーションの実装経験があり、さらに高度な構成を学びたいエンジニア
- エージェント型システムや RAG の設計に関心がある技術者
- Python の基礎知識があり、API を使ったプログラミング経験がある方

前提知識として、Python の基礎文法と、REST API の基本的な概念を理解していることを想定しています。
LLM や LangChain の経験は必須ではありませんが、あると理解がスムーズです。

## 本書の構成

本書は以下の3部構成になっています。

### 第1部: 基礎編（第1-3章）

エージェントと RAG の基本概念、LangChain と LangGraph のエコシステム、1.0 版の意義を解説します。
AI 活用の潮流と、なぜエージェントが重要なのかを理解できます。

### 第2部: 実装編（第4-6章）

LangGraph と LangChain のそれぞれの特徴と使い分けを、実装例とともに説明します。
グラフ構造によるワークフロー、ツール呼び出し、RAG との統合など、実践的な実装パターンを学びます。

### 第3部: 実践編（第7-12章）

ビジネス視点でのサービス企画から、エンジニアリング実践、運用時の落とし穴、将来展望までを扱います。
PoC から本番運用まで、実際のプロジェクトで必要となる知識を提供します。
