# イントロダクション：AI活用の潮流

## なぜ今、エージェントとRAGなのか

LLMとRAGは企業の情報活用における中核技術になりました。
ChatGPTをはじめとする生成AIの登場により、テキスト生成の品質は劇的に向上しました。
しかし単体のLLMには知識の鮮度と専門性の限界があります。
RAGは検索と生成を組み合わせてドメイン知識をLLMに供給します。
企業が保有する文書やデータベースと連携することで、LLMは最新かつ正確な情報をもとに回答できます。

エージェントはツール呼び出しと状態管理を通じて複雑なタスクを自律的に遂行します。
単なる質問応答を超えて、外部APIの呼び出しやデータベースの検索、計算処理などを組み合わせた業務フローを実現します。
人間の指示を受けて自律的に行動するエージェントは、業務効率化の鍵となります。

## 本書の狙いと読者対象

本書はビジネス価値の観点と実装の観点を往復しながら学習を進めます。
ビジネスパーソンにはエージェントとRAGがどのように価値を生み出すのかを理解してもらいます。
エンジニアには具体的な実装パターンとベストプラクティスを提供します。
両者が協力することで、実用的なAIアプリケーションを構築できます。

想定読者は以下の通りです。
AIを活用した新規事業や業務改善を検討しているビジネスパーソン。
LLMアプリケーションの実装経験があり、さらに高度な構成を学びたいエンジニア。
エージェント型システムやRAGの設計に関心がある技術者。
Python の基礎知識があり、API を使ったプログラミング経験がある方。

本書は以下の構成で展開します。

## 実装環境のセットアップ

実装準備としてPythonのセットアップとuvとVSCodeの導入を行います。
本書のサンプルコードは Python 3.12 以上を前提としています。
パッケージ管理には uv を使用し、開発環境には VSCode を推奨します。

### Python 3.12 のインストール

macOS の場合は Homebrew を使ってインストールします。

```bash
brew install python@3.12
```

Linux の場合は pyenv や apt などを使用してください。
Windows の場合は公式サイトからインストーラーをダウンロードします。

### uv のインストール

uv は Rust で書かれた高速な Python パッケージマネージャーです。
pip や poetry に比べて依存解決が高速で、プロジェクト管理が簡潔になります。

```bash
brew install uv  # macOS
# または
curl -LsSf https://astral.sh/uv/install.sh | sh  # Linux/macOS
```

### VSCode のセットアップ

VSCode は無料で使える高機能なエディタです。
Python 拡張機能をインストールすることで、コード補完やデバッグが快適に行えます。

VSCode をダウンロードしてインストールした後、以下の拡張機能を追加します。
Python（Microsoft 提供）
Pylance（Microsoft 提供、型チェックと補完強化）

## プロジェクトの初期化

プロジェクト骨格を雛形から作成して以後の章で段階的に拡張します。
サンプルコードは GitHub リポジトリの `tutorial/` ディレクトリに格納されています。

```bash
git clone https://github.com/yourusername/book-lc1.git
cd book-lc1/tutorial
uv python install 3.12
uv venv -p 3.12
source .venv/bin/activate  # macOS/Linux
# または
.venv\Scripts\activate  # Windows
uv sync
```

これで必要な依存パッケージがすべてインストールされます。
次章以降では、このプロジェクトを拡張しながら実装を進めます。
